    03 · K1
    什么是提示词
    理解"下教学任务"和"聊天"的本质区别
    提示词不是"和AI聊天"，而是给AI下一份清晰的教学任务书。就像你给实习生布置工作——说清"做什么、给谁用、要什么标准"，他才能交出你想要的东西。
    💡
    AI 就像一个能力很强、但完全不了解你班级的"新来的实习生"。你对他说"写个教案"，他会按自己的理解随便写一份；但如果你交待清楚年级、课文、学情、重点、格式，他就能交出你真正能用的东西。提示词，就是这份"任务交待书"。
    一句话记住：提示词的本质 = 把你脑子里的教学经验，翻译成AI能精确执行的任务指令。
    04 · K2
    什么是智能体 Agent
    理解为什么WorkBuddy不只是聊天机器人
    智能体比聊天机器人更进一步——它能记住上下文、串联多个步骤、主动调用工具。普通AI你问一句答一句；Agent你交一个目标，它自己拆解、执行、产出。
    💡
    普通聊天 AI 像"咨询客服"——你问一句他答一句，但他不会替你把事办了。而 Agent 像"全能助理"——你说"帮我准备下周的公开课"，他会自己拆成几步：先写教案、再做分层任务、最后生成PPT，一步一步把成品交到你手上。
    一句话记住：大模型是"脑"（负责思考生成），智能体是"手"（调用脑和各种工具去办事），同一个智能体可以接不同的"脑"。
    05 · K3
    什么是 Skills 技能
    理解AI如何一句话生成成品文件
    Skills是AI的"专业能力包"——每个 Skill 把"指令 + 模板 + 工具"打包，AI 需要时自动调用。装了 PPT 技能就会做课件，装了表格技能就会出 Excel。
    💡
    AI 本身像一个"什么都会一点、但样样不精通"的大学生。Skills 就是给他颁发的"教师资格证""PPT 工程师证""数据分析证"——每装一个 Skill，他就多一项拿得出手的专业本领，创建一次、永久复用。
    一句话记住：技能需要明确的触发词（如"做成PPT"）才会被调用。本次培训用的 PPT/Excel/Word/PDF 生成，就是内置的几个技能。
    06 · K4
    什么是知识库 RAG
    理解为什么上传教材后AI回答更准
    知识库让AI基于你的教材、课标、校本题来回答，而不是凭空编造。就像让AI先"读完"你的资料，再基于资料回答——准确率大幅提升。
    💡
    不建知识库直接问 AI，就像让学生"闭卷考试"——只能凭脑子里的记忆答，难免记错或编造（AI 的"幻觉"）。上传教材建好知识库后，相当于"开卷考试"——AI 可以翻着你给的书来回答，有据可查。
    一句话记住：RAG = 让AI"先读你的书，再回答你的问题"。上传教材后，AI的输出会贴合你的真实教学内容。
    07 · K5
    什么是龙虾 OpenClaw 助手
    你的数字分身：有灵魂、有记忆、持续进化
    龙虾助手是生活在本地或云端的智能助手——它有自己的灵魂和记忆，能持续学习你的习惯和偏好，像一个越用越懂你的数字分身。
    💡
    普通 AI 工具像"每次都换人的代课老师"——每次来都不认识你。而龙虾助手像一位"长期和你搭档的助教"——他记得你的每个学生、熟悉你的教学风格、知道你上次备课改到哪了，越合作越默契。
    一句话记住：龙虾助手 = 有记忆、有灵魂、持续进化的数字分身。它是AI从"工具"走向"伙伴"的下一步。
    08
    提示词八要素公式
    不是提问，是下任务
    角色 + 场景 + 目标 + 学情 + 材料 + 约束 + 输出格式 + 评价标准
    先说清给谁用（角色和对象）→ 再说清要解决什么教学问题（目标和学情）→ 最后说清输出成什么样（格式和评价标准）

    10
    WorkBuddy：通用智能体工作台
    不是单一教学工具，而是能帮老师执行复杂工作的 AI 同事

    ① 自然语言交任务

    老师不用先学复杂软件操作。把目标说清楚，WorkBuddy会理解需求、拆解步骤、持续执行。

    ② 资料与本地文件处理

    可围绕教材、课标、教案、学生反馈、成绩表等资料工作，用真实材料支持备课、分析和改写。

    ③ 多模型与专家协同

    面向不同任务调用不同模型和专家能力：调研、内容生成、数据分析、教学设计可以并行推进。

    ④ Skills 与连接器扩展

    通过技能和生态连接，把对话延伸到PPT、Word、Excel、图片、知识库和办公协作工具。

    ⑤ 可验证的教学工作流交付

    对老师最有价值的不是“聊得像”，而是交付可检查、可修改、可带走的成果：教案、课件、任务单、错题表、学情报告。教师仍负责判断与把关。

    优化你的第一版

  • 上传教材让AI改进
    优化指令 · 点击复制

  • 直接说不满意的地方

    16 · P7
    体验设计一个新 Skill
    案例：新课标教案 + 分层任务单
    Skill 不是一句提示词。它要把重复任务拆成：触发方式、输入模板、执行规则、输出模板、质量检查。老师今天体验的不是“让 AI 写教案”，而是把一套备课方法沉淀成可复用能力。
    模块“新课标教案+分层任务单”Skill 设计
    触发词用“新课标教案 Skill”帮我设计一课时教案和三层任务单
    输入模板年级学科、教材版本、课题、课标/核心素养、班情、重难点、课时、教材页或资料
    执行规则缺关键资料先追问；不编造教材内容；目标、活动、评价、分层任务必须互相对应
    输出模板基本信息、学习目标、重难点、教学流程、三层任务单、评价量规、教师检查清单
    测试指令用这个 Skill 处理“四年级语文《爬山虎的脚》”，检查输出是否可直接修改使用
    创建 Skill 的完整提示词 · 点击复制
    请帮我创建一个名为“新课标教案+分层任务单”的教学 Skill。 【触发方式】 当我说“用新课标教案 Skill”时，请自动按下面流程工作。 【必须收集的输入】 年级学科、教材版本、课题、课标目标或核心素养、本班学情、已有基础、常见困难、课时长度、教材页/资料。 如果缺少课题、年级学科、学情、课时这四项，请先追问，不要直接生成。 【执行规则】 1. 不编造教材细节；资料不足时标注“基于当前信息的假设”。 2. 学习目标要可观察、可评价，不写空泛口号。 3. 教学活动必须对应目标，问题串要递进。 4. 分层任务分为基础、发展、挑战三层，不降低基础层的尊严感。 5. 输出最后必须给教师检查清单，提醒我审核准确性和班级适配性。 【输出模板】 一、基本信息 二、学习目标：知识理解、能力方法、核心素养 三、重点难点 四、教学流程表：环节、教师活动、学生活动、关键问题、评价方式、用时 五、分层任务单：基础层、发展层、挑战层；每层含目标、任务、提示、评价标准、预计用时 六、评价量规 七、教师检查清单 请先把这份 Skill 说明书整理出来，再给我一个测试用例。
    关键体验：老师不是只复制提示词，而是在定义“AI每次应该如何稳定工作”。这就是把个人备课经验变成团队可复用资产。
    17 · P8
    打造教研智能体：用 vibe coding
    培训最高潮——从一份成品，到一个持续进化的教研体系
    vibe coding = 用说话的方式指挥AI写代码、搭工具，不用懂编程语法。
    教研智能体 = 校本知识库 + 教研组技能 + 持续记忆。让组里任何人问一句，都能得到团队经验的积累。
    vibe coding指令：搭建教研智能体 · 点击复制
    请帮我打造一个"教研智能体"，具备以下能力： 1.【校本知识库】我会陆续上传教研组的优质教案、易错题、评价量规，请基于这些资料回答，并注明依据来源，没有就说"资料库暂无"，不要编造。 2.【教研组技能】请接入我的"分层出题""教案审核"等专属技能。 3.【持续记忆】请记住我们组的学科年级、学生薄弱点、偏好风格。 4.【工作流程】当我说"备这节课"时，自动：找参考→生成教案→审核→给可下载成品。 请先确认理解，然后告诉我该怎么把资料喂给你。
    关键认知：智能体的水平取决于你喂的内容质量。持续沉淀好课、好题、好量规，它才会越来越聪明——这就是校本知识库的价值。

    成品化指令速查
    先用对话把内容做对，再用技能把成品做出来
    教案 → Word

    错题分析 → Excel
    请把错题分析结果做成一份Excel表格，包含题号、知识点、错误率、错因、补救建议、变式题
    教案 → 教学PPT
    请基于以上教案内容，做成一份15页的教学PPT，要有标题页、目标页、每个环节一页、总结页
    学情报告 → PDF
    请把学情分析整理成一份PDF报告，含班级整体情况、薄弱点排名、分层建议

    六条红线 + 学生AI公约
    AI在旁边，教师在中间，学生在活动中
    风险怎么做
    学生隐私上传文件前先脱敏——去掉学生姓名、身份证号
    AI会出错AI生成的知识、解析、答案必须审核，不能直接用
    评价公平AI的评分只能做参考，正式评分权在教师手中
    学生依赖学生端AI要给提示不给答案，教学生先想再问
    版权合规只上传授权资料，生成内容注明AI辅助
    课堂关系AI在旁边，教师在中间，学生在活动中
