    第三步 · 28-40分钟 认识WorkBuddy这个通用智能体工作台

    ① 自然语言交任务

    把目标说清楚，WorkBuddy会理解需求、拆解步骤、持续执行，而不是只回答一句建议。

    ② 资料与本地文件处理

    围绕教材、课标、教案、学生反馈、成绩表等真实资料工作，支持备课、分析和改写。

    ③ 多模型与专家协同

    不同任务调用不同模型和专家能力，调研、内容生成、数据分析、教学设计可以并行推进。

    ④ Skills 与连接器扩展

    通过技能和生态连接，把对话延伸到PPT、Word、Excel、图片、知识库和办公协作工具。

    ⑤ 可验证的教学工作流交付

    对老师最有价值的是交付可检查、可修改、可带走的成果：教案、课件、任务单、错题表、学情报告。教师仍负责判断与把关。

    第五步 · 0-10分钟 生成你的第一版
    跟着讲师一起做——复制提示词模板，填入你的真实信息，发送给AI。

  • 复制提示词到WorkBuddy对话框

    点击提示词旁边的"复制"按钮，然后粘贴到WorkBuddy的对话框里。

  • 替换方括号里的内容

    第六步 · 10-25分钟 优化你的第一版
    这一步你要做的：让AI根据你的真实素材和反馈来改进第一版。

    怎么优化？三种方式

  • 上传教材让AI改进
    优化指令 · 点击复制
    请基于我上传的教材内容，改进上面的教案，确保贴合教材的重难点和知识点
  • 补充学情信息
    优化指令 · 点击复制
    我的学生情况补充：[比如"本班有3名学生基础很薄弱，需要额外关注；大部分学生能跟上但缺乏主动思考习惯"]。 请根据这个学情调整教学设计。
  • 直接说不满意的地方
    修改指令 · 点击复制
    以下地方需要修改： 1. [比如"导入部分太长了，请缩短到2分钟内"] 2. [比如"分层任务的基础层太简单了，请适当提高难度"] 3. [比如"评价标准太笼统，请给出具体的评分标准"] 请修改后重新输出。
    • 记住：先求"能用"，不要卡在"完美"。不满意就跟AI说，这就是迭代。
    第七步 · 25-40分钟 体验设计一个新 Skill：新课标教案+分层任务单
    这一步不是让AI临时写一份教案。你要体验如何把一套备课方法设计成 Skill，让AI以后每次都按同一套标准工作。
    什么是 Skill 设计？把重复任务拆成“触发方式、输入模板、执行规则、输出模板、质量检查”。设计好之后，你只要说一句“用新课标教案 Skill”，AI就知道该问什么、怎么做、按什么格式交付。

    这个 Skill 要解决什么问题？

  • 减少重复交代——不用每次都重新说明“要有课标、学情、流程、分层任务、评价量规”
  • 提高输出稳定性——每次都按同一套输入字段和输出结构生成，便于老师审核
  • 沉淀团队经验——教研组可以共同打磨一套适合本校的备课标准

    • Skill 说明书

    模块本案例怎么设计
    技能名称新课标教案+分层任务单
    触发方式用“新课标教案 Skill”帮我设计一课时教案和三层任务单
    输入模板年级学科、教材版本、课题、课标/核心素养、班情、已有基础、常见困难、课时、教材页或资料
    执行规则缺关键资料先追问；不编造教材内容；目标、活动、评价、分层任务必须互相对应
    输出模板基本信息、学习目标、重难点、教学流程、三层任务单、评价量规、教师检查清单

    实操一：复制下面的提示词，创建 Skill

    创建 Skill：新课标教案+分层任务单 · 点击复制
    请帮我创建一个名为“新课标教案+分层任务单”的教学 Skill。 【触发方式】 当我说“用新课标教案 Skill”时，请自动按下面流程工作。 【必须收集的输入】 年级学科、教材版本、课题、课标目标或核心素养、本班学情、已有基础、常见困难、课时长度、教材页/资料。 如果缺少课题、年级学科、学情、课时这四项，请先追问，不要直接生成。 【执行规则】 1. 不编造教材细节；资料不足时标注“基于当前信息的假设”。 2. 学习目标要可观察、可评价，不写空泛口号。 3. 教学活动必须对应目标，问题串要递进。 4. 分层任务分为基础、发展、挑战三层，不降低基础层的尊严感。 5. 输出最后必须给教师检查清单，提醒我审核准确性和班级适配性。 【输出模板】 一、基本信息 二、学习目标：知识理解、能力方法、核心素养 三、重点难点 四、教学流程表：环节、教师活动、学生活动、关键问题、评价方式、用时 五、分层任务单：基础层、发展层、挑战层；每层含目标、任务、提示、评价标准、预计用时 六、评价量规 七、教师检查清单 请先把这份 Skill 说明书整理出来，再给我一个测试用例。

    实操二：用一个测试任务验证 Skill

    测试指令 · 点击复制
    请用“新课标教案 Skill”处理下面这个任务： 年级学科：小学四年级语文 教材版本：统编版 课题：《爬山虎的脚》 本班学情：学生能读懂课文大意，但观察性语言积累弱，表达时容易空泛 课时：40分钟 重点：理解作者连续细致观察的方法 输出：一课时教案 + 三层任务单 + 评价量规
    测试时看三件事：
  • 它是否先检查输入是否完整
  • 目标、活动、评价和分层任务是否互相对应
  • 输出是否能被老师直接审核和修改，而不是泛泛建议
    • 重要提醒：Skill 的价值不在于一次生成，而在于稳定复用。第一次结果不好，就继续改 Skill 说明书，而不是只改这一份教案。
    第八步 · 40-53分钟 打造教研智能体：用vibe coding建你的校本知识库
    这是整场培训的最高潮，也是你迈向"AI教研合伙人"的起点。前7步你学会了用对话、知识库、技能做单份成品——现在，用vibe coding把它们串联成一个能持续服务教研组的智能体。
    什么是 vibe coding？就是用说话的方式指挥AI写代码、搭工具。你不用懂任何编程语法，只需用自然语言告诉AI"我要一个什么样的工具"，它就帮你实现。今天你体验的WorkBuddy、龙虾助手，背后都是这样"说"出来的。

    什么是"教研智能体"？

    教研智能体 = 校本知识库 + 教研组技能 + 持续记忆。它把组里的优质教案、易错题、评价量规、学情数据沉淀成AI能调用的知识，让组里任何人问一句，都能得到团队经验的积累——而且越用越聪明。

  • 校本知识库（喂给它经验）

    把教研组的优质教案、往届易错题、评价量规上传，让AI基于这些"团队资产"回答，而不是凭空编造。

  • 教研组技能（让它按标准做事）

    把第七步设计的专属技能（教案审核、分层出题等）接入，让智能体每次都按组里的统一标准执行。

  • vibe coding（用说话搭建）

    用自然语言告诉AI你要什么功能，它帮你把知识库、技能、流程串成一个完整的教研助手。

    • 实操：用vibe coding打造你的第一个教研智能体

    复制下面的指令到WorkBuddy，按提示把你的教研组资产喂给它：

    vibe coding指令：搭建教研智能体 · 点击复制
    请帮我打造一个"教研智能体"，它要具备以下能力： 1. 【校本知识库】我会陆续上传我们教研组的优质教案、易错题统计、评价量规。 请基于这些资料回答问题，并在回答时注明依据来自哪份资料。 如果资料里没有相关内容，请明确说"资料库中暂无"，不要编造。 2. 【教研组技能】请接入以下我之前设计的专属技能（如果没有，先帮我创建）： - "分层出题"：按我们组的标准出基础/发展/挑战三层练习 - "教案审核"：按六维度审核教案质量 3. 【持续记忆】请记住： - 我们组的学科和年级：[填写，如"小学数学中年级"] - 我们学生的普遍薄弱点：[填写] - 我们偏好的教案风格：[填写] 4. 【工作流程】当我说"备这节课"时，请自动： 先从知识库找相关优质教案参考 → 按我们的风格生成新教案 → 用审核技能检查 → 最后给我可下载的成品 请先确认你理解了，然后告诉我接下来该怎么把资料喂给你。
    搭建好后，你可以这样用它：
  • "备三年级《分数的初步认识》" → 它自动走完整流程，给你一份基于本校经验的教案
  • "这道错题为什么学生总错？" → 它从知识库找往届同类错题，给错因分析和变式题
  • "帮新老师快速上手我们组的备课标准" → 它基于组里沉淀的教案给出示范
    • 关键认知：教研智能体的水平，取决于你喂给它的内容质量。垃圾进、垃圾出。所以持续把组里的好课、好题、好量规沉淀进去，它才会越来越聪明——这就是"校本知识库"的价值：让团队经验不随人员流动而流失。

    🎉 你已经完成了全部13关！

    从一个提示词，到一份教学成品，再到一个教研智能体——你现在拥有的不只是一个工具，而是一套可持续进化的教学能力体系。

    展示与收获

    技能使用指南

    技能不是文件格式按钮，而是把高频教学需求沉淀成可复用工作流

    核心理念：先把真实需求讲清楚，再把重复任务沉淀成 Skill。
  • 看需求是否高频——多位老师反复需要，才值得做成 Skill
  • 看输入是否稳定——年级、课题、班情、数据、材料等信息能被标准化填写
  • 看输出是否可审核——教案、任务单、学情报告等成果必须由老师检查后使用

    • 关键提醒

    提醒说明
    不是所有需求都要做成 Skill一次性问题适合直接对话；高频、稳定、可复用的任务才适合沉淀
    输入要标准化一个好 Skill 要明确让老师填什么，例如课题、班情、教材、学生数据
    输出要能落地成果要能被老师检查、修改、带进课堂，而不是只给一段泛泛建议
    教师仍然把关Skill 提高效率，但教学判断、内容准确性和课堂适配仍由教师负责

    安全使用须知

    六条红线，必须遵守

    风险怎么做
    学生隐私上传文件前先脱敏——去掉学生姓名、身份证号、家庭信息
    AI会出错AI生成的知识、解析、答案必须审核，不能直接用
    评价公平AI的评分只能做参考，正式评分权在教师手中
    学生依赖学生端AI要给提示不给答案，教学生先想再问
    版权合规只上传授权资料，生成内容注明AI辅助
    课堂关系AI在旁边，教师在中间，学生在活动中
