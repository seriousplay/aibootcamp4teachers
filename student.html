    第一步 · 8-18分钟 提示词入门：不是提问，是下任务
    你要学会的：用一句话给AI下一个清晰的教学任务，而不是泛泛地问。

    提示词八要素公式

    记住这个公式
    角色 + 场景 + 目标 + 学情 + 材料 + 约束 + 输出格式 + 评价标准

    看看区别

    📋 写教案

    ✗ 太笼统
    "帮我写一个教案"
    ✓ 够具体
    "你是一名小学语文教研员。请基于四年级上册《爬山虎的脚》一课，结合本班学生识字基础偏弱、课堂时间40分钟、要突出分层和课堂活动，输出学习目标、问题链、分层任务、板书设计和评价量规。"

    📊 分析错题

    ✗ 太笼统
    "帮我分析错题"
    ✓ 够具体
    "你是一名数学学情分析助手。以下是五年级10名学生'异分母分数加减法'的答题统计……请输出：共性错误排序、错因假设、3分钟讲解稿、2道例题、3道变式题、分层补救建议。"
    三句话记住怎么写提示词：
    1. 先说清给谁用——角色和对象
    2. 再说清要解决什么教学问题——目标和学情
    3. 最后说清输出成什么样——格式和评价标准
    第二步 · 18-28分钟 首次上手：跟着讲师操作
    这是你第一次操作WorkBuddy的环节。跟着讲师投屏一步一步做，不用怕出错。三步操作完成后，你就基本会用这个平台了。

    操作一：发第一条消息（3分钟）

    在对话框里输入下面这段话，把方括号里的内容换成你自己的真实信息，然后发送：

    第一条消息 · 点击复制
    你好，我是一名[小学语文/数学/英语……]老师，教[年级]。 今天来学习用AI辅助教学。 请用一句话告诉我你能帮我做什么。
    发送后等AI回复——这就是你的第一次AI对话！如果没收到回复，举手找讲师帮忙。

    操作二：上传一个文件（3分钟）

  • 找到上传按钮（讲师会指给你看）
  • 用手机拍一页你正在用的教材，或者上传提前准备好的教材PDF/图片
  • 输入下面这句话发送：
    • 上传文件后的指令 · 点击复制
    请帮我看看这个文件的内容，用3句话总结
    恭喜！你已经掌握了WorkBuddy的全部基础操作：打字说话、上传文件、看结果。下半场就用这三个动作来完成你的教学成品。
    第三步 · 28-40分钟 认识WorkBuddy的五种能力
    这一步你只需要看和听。讲师会演示WorkBuddy能做什么，你重点理解每种能力能帮你解决什么教学问题。

    ① 对话即任务

    你说什么，AI就做什么。不需要搭建，开口就说。

    ② 文件即知识库

    上传教材PDF，AI就基于你的教材来回答，不凭空编造。

    ③ 一段话串流程

    "先分析错题，再生成讲解，最后出练习"——一句话，AI自动分步完成。

    ④ 成品直接出

    不只是聊天文字，AI能直接生成可下载的文件。

    ⑤ 技能即工具箱（重点！）

    内置PPT/Excel/Word/PDF/图片等技能，一句话就能生成可下载的文件成品。这是你下半场要用的核心能力。

    第五步 · 0-10分钟 生成你的第一版
    跟着讲师一起做——复制提示词模板，填入你的真实信息，发送给AI。
  • 找到你的任务提示词

    如果你已经选了任务，点击"我的任务提示词"查看你的专属模板。

  • 复制提示词到WorkBuddy对话框

    点击提示词旁边的"复制"按钮，然后粘贴到WorkBuddy的对话框里。

  • 替换方括号里的内容

    把 [年级]、[课题]、[学情] 等方括号里的内容，替换成你自己的真实信息。

  • 发送，等AI生成结果

    点击发送，等待AI生成。通常几秒到十几秒就能出结果。

    • 完成这一步后：你已经有了AI生成的第一版内容。先不要急着改——通读一遍，标记满意和不满意的地方。
    第六步 · 10-25分钟 优化你的第一版
    这一步你要做的：让AI根据你的真实素材和反馈来改进第一版。

    怎么优化？三种方式

  • 上传教材让AI改进
    优化指令 · 点击复制
    请基于我上传的教材内容，改进上面的教案，确保贴合教材的重难点和知识点
  • 补充学情信息
    优化指令 · 点击复制
    我的学生情况补充：[比如"本班有3名学生基础很薄弱，需要额外关注；大部分学生能跟上但缺乏主动思考习惯"]。 请根据这个学情调整教学设计。
  • 直接说不满意的地方
    修改指令 · 点击复制
    以下地方需要修改： 1. [比如"导入部分太长了，请缩短到2分钟内"] 2. [比如"分层任务的基础层太简单了，请适当提高难度"] 3. [比如"评价标准太笼统，请给出具体的评分标准"] 请修改后重新输出。
    • 记住：先求"能用"，不要卡在"完美"。不满意就跟AI说，这就是迭代。
    第七步 · 25-40分钟 设计你的专属技能：把经验变成可复用的"AI助手"
    这是整场培训最让你惊喜的一步。你已经会用平台内置的技能了——现在，把你自己的专业经验，设计成一个专属技能，让AI每次都按你的标准做事。
    什么是"专属技能"？把你反复在用的教学经验（比如你的教案审核标准、你的分层出题套路、你的家长信风格），写成一套清晰的指令模板。写好之后，以后一句话就能触发，AI自动按你的标准执行——不用每次都重新描述一遍。

    为什么要设计专属技能？

  • 把经验固化下来——你的教学智慧不再只存在脑子里，而是变成AI能反复执行的标准流程
  • 一次设计，永久复用——设计好后，每次备课、出题、写文案都能一键调用
  • 分享给团队——把好技能分享给同组、同校老师，让整个教研组都用同一套高质量标准

    • 设计专属技能的三步法

  • 找到你反复在做的事

    想想哪些教学任务你每次都要重复交代同样的要求？比如"分层出题总是要强调基础5题+发展4题+挑战3题"。

  • 把要求写成清晰的指令模板

    用提示词的方式，把你的标准、流程、输出格式写成一个完整的"技能说明书"。

  • 给技能起个名字，以后一句话触发

    比如命名为"我的分层出题"，以后只需说"用我的分层出题技能"，AI就按你的标准执行。

    • 实操：设计你的第一个专属技能

    从下面挑一个最贴合你的场景，复制对应模板到WorkBuddy，按提示补全你的标准：

    技能模板1：我的教案审核标准 · 点击复制
    请帮我创建一个名为"我的教案审核"的专属技能。 这个技能的作用：每次我上传或粘贴一份教案，你都要按以下我的专属标准审核： 我的审核标准（请帮我补全并细化）： 1. 学习目标：必须包含[新课标核心素养，请填写你学科的具体素养] 2. 学情匹配：必须考虑[本班学生的薄弱点，请填写] 3. 分层任务：必须有基础/发展/挑战三层，且[我的分层要求，请填写] 4. 评价标准：必须[可量化/可操作，我的要求是：请填写] 请把这套标准整理成一份清晰的技能说明书，以后我叫"审核教案"你就按这个执行。
    技能模板2：我的分层出题套路 · 点击复制
    请帮我创建一个名为"我的分层出题"的专属技能。 这个技能的作用：我说一个知识点，你就按我固定的套路出题： 我的出题套路： - 基础层：[5]道基础题，覆盖核心知识点，[我的基础题要求：请填写] - 发展层：[4]道应用题，需要迁移理解，[我的发展题要求：请填写] - 挑战层：[3]道综合题，需要多步推理，[我的挑战题要求：请填写] - 每道题附答案和解析 - 难度要[适合我班学生的水平，请填写] 请把这套标准整理成技能说明书，以后我说"分层出题"你就按这个执行。
    技能模板3：我的家长信风格 · 点击复制
    请帮我创建一个名为"我的家长信"的专属技能。 这个技能的作用：每次我告诉你一件事，你就帮我写一封符合我个人风格的家长信： 我的风格要求： - 语气：[温暖、鼓励、不贴负面标签，请按需调整] - 结构：先说事实，再说进步，最后给一个具体的下一步建议 - 字数：[家长信不超过300字 / 微信群消息不超过100字，请选择] - 绝不：[给学生贴标签 / 用命令语气，请补充你的底线] 请把这套风格整理成技能说明书，以后我说"写家长信"你就按这个执行。
    设计完专属技能后，你可以：
  • 马上测试它——说一句"用我的XX技能"，看AI是否按你的标准执行
  • 持续优化它——不满意就告诉AI"调整我的XX技能，把XX标准改成XX"
  • 分享给同事——把你设计好的技能说明发给同组老师，他们也能用
    • 重要提醒：专属技能是你教学经验的沉淀，它越用越准。但记住——AI执行的是你定的标准，标准本身的质量决定了产出的质量。所以设计技能时，把你的专业判断写清楚、写具体。
    第八步 · 40-53分钟 打造教研智能体：用vibe coding建你的校本知识库
    这是整场培训的最高潮，也是你迈向"AI教研合伙人"的起点。前7步你学会了用对话、知识库、技能做单份成品——现在，用vibe coding把它们串联成一个能持续服务教研组的智能体。
    什么是 vibe coding？就是用说话的方式指挥AI写代码、搭工具。你不用懂任何编程语法，只需用自然语言告诉AI"我要一个什么样的工具"，它就帮你实现。今天你体验的WorkBuddy、龙虾助手，背后都是这样"说"出来的。

    什么是"教研智能体"？

    教研智能体 = 校本知识库 + 教研组技能 + 持续记忆。它把组里的优质教案、易错题、评价量规、学情数据沉淀成AI能调用的知识，让组里任何人问一句，都能得到团队经验的积累——而且越用越聪明。

  • 校本知识库（喂给它经验）

    把教研组的优质教案、往届易错题、评价量规上传，让AI基于这些"团队资产"回答，而不是凭空编造。

  • 教研组技能（让它按标准做事）

    把第七步设计的专属技能（教案审核、分层出题等）接入，让智能体每次都按组里的统一标准执行。

  • vibe coding（用说话搭建）

    用自然语言告诉AI你要什么功能，它帮你把知识库、技能、流程串成一个完整的教研助手。

    • 实操：用vibe coding打造你的第一个教研智能体

    复制下面的指令到WorkBuddy，按提示把你的教研组资产喂给它：

    vibe coding指令：搭建教研智能体 · 点击复制

  • "备三年级《分数的初步认识》" → 它自动走完整流程，给你一份基于本校经验的教案
  • "这道错题为什么学生总错？" → 它从知识库找往届同类错题，给错因分析和变式题
  • "帮新老师快速上手我们组的备课标准" → 它基于组里沉淀的教案给出示范
    • 关键认知：教研智能体的水平，取决于你喂给它的内容质量。垃圾进、垃圾出。所以持续把组里的好课、好题、好量规沉淀进去，它才会越来越聪明——这就是"校本知识库"的价值：让团队经验不随人员流动而流失。

    技能使用指南

    技能是WorkBuddy最重要的能力——一句话生成可下载文件成品

    核心理念：先用对话把内容做对，再用技能把成品做出来。
  • 先把内容用对话生成——教案、错题分析、PBL方案等
  • 告诉AI你要什么格式——"请把以上内容做成PPT/Excel/Word/PDF"
  • 等AI生成完成，下载文件，打开检查——不满意就告诉AI改哪里

    • 关键提醒

    提醒说明
    先有内容，再出成品技能是把已有内容转成文件。先在对话中把内容写好，再调用技能出成品
    一份内容，多种格式同一份教案可以生成Word版存档、PPT版上课、PDF版分享
    生成后必须检查下载后打开看：内容对不对？格式好不好？需要手动调整吗？
    不满意就追加指令"PPT再加一页板书设计""Excel多加一列难度等级"——直接说

    安全使用须知

    六条红线，必须遵守

    风险怎么做
    学生隐私上传文件前先脱敏——去掉学生姓名、身份证号、家庭信息
    AI会出错AI生成的知识、解析、答案必须审核，不能直接用
    评价公平AI的评分只能做参考，正式评分权在教师手中
    学生依赖学生端AI要给提示不给答案，教学生先想再问
    版权合规只上传授权资料，生成内容注明AI辅助
    课堂关系AI在旁边，教师在中间，学生在活动中
